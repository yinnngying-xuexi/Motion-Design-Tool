# DataMotion 可视化大屏动效编辑器 PRD

## 1. 文档信息

- 产品名称：DataMotion
- 文档版本：V0.1 高保真原型确认版
- 当前阶段：静态 Vue 高保真原型
- 适用范围：动效编辑首页 MVP
- 设计规范：`doc/design.md` 中的 Glassline 规范

## 2. 产品定位

DataMotion 是一款面向可视化大屏 UI 设计师的 CSS 动效编辑工具。

设计师可以在页面中选择常用动效模板，查看实时预览，调整动效参数，并导出 CSS 或 JSON 配置，交付给前端开发继续实现。

当前版本重点不是完整开发系统，而是先确认核心页面结构、交互方式、信息层级和视觉风格。

## 3. 目标用户

核心用户：

- 可视化大屏 UI 设计师

次要用户：

- 前端开发人员
- 产品经理
- 交互设计师
- 项目负责人

## 4. 用户问题

当前大屏动效交付中存在以下问题：

- 设计稿很难准确表达动效速度、延迟、幅度和循环方式。
- UI 设计师不方便直接修改 CSS 动效参数。
- 动效主要依赖前端实现，沟通成本高。
- 同一项目中动效风格容易不统一。
- 前端实现后需要多次反复调整。

## 5. 当前原型目标

本阶段原型需要完成以下目标：

- 建立动效编辑器的核心页面框架。
- 验证四区布局是否适合设计师工作流。
- 验证动效选择、参数编辑、实时预览、代码导出的主流程。
- 按 Glassline 设计规范确定视觉方向。
- 为后续正式 Vue 开发提供产品依据。

## 6. 当前原型范围

包含：

- 顶部工具栏
- 左侧动效库
- 中央实时预览画布
- 右侧参数编辑面板
- 底部动效层
- CSS / JSON 导出区域
- 保存动效弹窗

暂不包含：

- 后端服务
- 用户登录和权限
- 真实文件保存
- 真实团队协作
- 完整时间轴编辑器
- 可视化 keyframes 编辑器
- Figma 插件
- React / Vue 组件正式导出

## 7. 页面结构

### 7.1 顶部工具栏

用途：展示产品身份、当前动效和全局操作。

当前包含：

- 产品名称 DataMotion
- 当前动效名称
- 画布尺寸选择
- 撤销按钮
- 重做按钮
- 重置按钮
- 保存动效按钮

交互要求：

- 保存动效是当前主页面唯一高强调操作。
- 重置用于恢复当前动效默认参数。
- 画布尺寸用于切换预览规格。

### 7.2 左侧动效库

用途：让用户选择基础 CSS 动效。

当前包含：

- 动效搜索
- 分类筛选
- 动效卡片列表
- 动效名称
- 推荐使用场景
- 动效分类标签
- 可编辑参数数量
- 收藏按钮
- 小型动效预览

当前分类：

- 全部
- 入场动效
- 循环动效
- 强调动效
- 告警动效
- 大屏装饰
- 我的动效
- 项目动效库

当前内置动效：

- 淡入
- 上滑进入
- 左滑进入
- 缩放进入
- 呼吸
- 上下浮动
- 轻微闪烁
- 慢速旋转
- 放大提示
- 高亮发光
- 边框高亮
- 红色闪烁
- 脉冲扩散
- 边框流光
- 扫描线
- 地图光点闪烁

交互要求：

- 点击动效卡片后，中央预览立即切换。
- 当前选中动效需要有明确选中状态。
- 右侧参数面板同步展示该动效支持的参数。
- 底部代码区同步生成 CSS / JSON。

### 7.3 中央实时预览画布

用途：展示当前动效应用后的实际效果。

当前包含：

- 播放
- 暂停
- 循环
- 预览对象切换
- 背景样式切换
- 当前画布规格展示
- 当前预览对象展示
- 大屏组件模拟画面

当前预览对象：

- 数据卡片
- 指标数字
- 图标
- 告警标签
- 图表容器
- 地图点位
- 边框面板
- 普通按钮
- 标题文字

当前背景样式：

- 雾灰底色
- 深墨底色
- 透明背景
- 白色底色

交互要求：

- 修改参数后，预览效果即时更新。
- 点击播放、暂停、循环后，预览状态同步变化。
- 切换预览对象后，动效仍可正常应用。
- 预览画面需保持高保真，但不要求模拟真实大屏业务数据。

### 7.4 右侧参数编辑面板

用途：让设计师以可视化方式编辑 CSS 动效参数。

当前包含：

- 当前动效名称
- 动效说明
- 重播按钮
- 参数控件
- 单个参数重置
- 动效冲突提示

当前参数类型：

- 数值滑块
- 数字输入框
- 颜色选择器
- 文本颜色值输入
- 下拉选择器

当前支持参数：

- 动效时长 duration
- 延迟时间 delay
- 播放次数 iteration
- 缓动曲线 easing
- 透明度 opacity
- 横向位移 translateX
- 纵向位移 translateY
- 缩放 scale
- 旋转 rotate
- 发光强度 glow
- 扩散半径 radius
- 边框宽度 borderWidth
- 动效颜色 color

交互要求：

- 参数变化后，预览区同步更新。
- 参数变化后，CSS / JSON 同步更新。
- 单个参数可重置。
- 全部参数可通过顶部重置按钮恢复默认值。
- 当动效层存在明显 transform 叠加时，展示冲突提示。

### 7.5 底部动效层

用途：展示当前元素上应用的动效组合。

当前包含：

- 动效层列表
- 启用 / 关闭动效层
- 复制动效层
- 删除动效层
- 添加动效
- 延迟时间展示
- 是否循环展示

当前默认动效层：

- 上滑进入
- 呼吸

交互要求：

- 用户可添加当前选中动效为新图层。
- 用户可关闭、复制、删除图层。
- 关闭图层后，该图层视觉上变为弱化状态。
- 当前阶段只验证轻量动效层概念，不做完整时间轴。

### 7.6 CSS / JSON 导出区域

用途：让用户查看并复制当前动效代码。

当前包含：

- CSS 标签
- JSON 标签
- 代码预览
- 复制代码按钮

交互要求：

- 切换 CSS / JSON 时，代码区内容同步切换。
- 点击复制后展示反馈状态。
- CSS 内容包含 class、animation、keyframes、transform、opacity、box-shadow 等信息。
- JSON 内容包含动效 ID、名称、分类、场景、参数、class 名称和来源。

### 7.7 保存动效弹窗

用途：模拟将当前动效保存为自定义模板。

当前包含：

- 动效名称
- 保存位置
- 使用说明
- 确认保存按钮
- 关闭按钮

保存位置：

- 我的动效
- 项目动效库

交互要求：

- 点击保存动效打开弹窗。
- 默认填入当前动效名称和描述。
- 确认保存后，新增动效进入对应分类。
- 当前阶段使用前端内存模拟保存，不做持久化。

## 8. 核心用户流程

### 8.1 选择并编辑动效

1. 用户打开 DataMotion。
2. 在左侧动效库选择一个动效。
3. 中央画布立即播放该动效。
4. 右侧展示该动效支持的参数。
5. 用户调整参数。
6. 预览画布和代码区同步更新。

### 8.2 导出代码

1. 用户调整好动效效果。
2. 在底部代码区选择 CSS 或 JSON。
3. 点击复制代码。
4. 系统展示复制反馈。
5. 用户将代码交付给前端开发。

### 8.3 保存为自定义动效

1. 用户点击顶部保存动效。
2. 系统打开保存弹窗。
3. 用户填写动效名称、保存位置和说明。
4. 点击确认保存。
5. 系统将新动效加入对应分类。

## 9. 设计规范

当前页面必须遵循 `doc/design.md` 中的 Glassline 设计规范。

### 9.1 色彩

- Primary：`#0F1419`，用于核心文字和深色预览画面。
- Secondary：`#4A5568`，用于说明文字、元信息和弱化内容。
- Tertiary：`#2C5EF5`，作为唯一强调色，用于主要操作和动效强调。
- Neutral：`#F1F3F5`，作为页面底色。
- Surface：`#FFFFFF`，作为卡片和面板底色。

### 9.2 字体

- 标题：Geist，600 weight。
- 正文：Geist，0.95rem。
- 标签：Geist Mono，0.75rem。

### 9.3 视觉原则

- 不使用渐变。
- 不引入额外强调色。
- 保持冷静、干净、工具化的界面气质。
- 让雾灰背景和白色面板承担主要空间层级。
- 每个屏幕只保留一个高强调主操作。

## 10. 数据结构建议

### 10.1 MotionTemplate

```ts
interface MotionTemplate {
  id: string;
  name: string;
  category: string;
  scene: string;
  description: string;
  previewType: 'slide' | 'pulse' | 'glow' | 'scan';
  className: string;
  params: MotionParam[];
}
```

### 10.2 MotionParam

```ts
interface MotionParam {
  key: string;
  label: string;
  type: 'range' | 'select' | 'color';
  default: string | number;
  min?: number;
  max?: number;
  step?: number;
  unit?: string;
  options?: Array<{
    label: string;
    value: string;
  }>;
}
```

### 10.3 MotionLayer

```ts
interface MotionLayer {
  uid: number;
  name: string;
  delay: number;
  loop: boolean;
  enabled: boolean;
}
```

## 11. 验收标准

### 11.1 页面验收

- 页面整体视觉符合 Glassline 规范。
- 不出现旧版森林绿、铜色或旧字体风格。
- 页面不使用渐变。
- 四区布局清晰：动效库、预览画布、参数面板、底部动效层和代码区。

### 11.2 交互验收

- 点击任意动效卡片后，预览和参数面板同步更新。
- 参数调整后，预览效果同步变化。
- 参数调整后，CSS / JSON 同步变化。
- 播放、暂停、循环按钮状态可切换。
- 预览对象和背景样式可切换。
- 动效层可添加、启用、关闭、复制、删除。
- 保存弹窗可打开、关闭、提交。
- 复制代码按钮有状态反馈。

### 11.3 内容验收

- 动效库至少包含当前 16 个基础动效。
- 每个动效有名称、分类、使用场景和说明。
- 每个动效有默认参数。
- 生成 CSS 可作为前端实现参考。
- 生成 JSON 能表达当前动效配置。

## 12. 后续开发建议

第一阶段正式开发建议：

- 将当前静态 Vue CDN 原型迁移为 Vite + Vue + TypeScript 项目。
- 将动效模板拆分为独立配置文件。
- 将 CSS 生成逻辑拆分为 generator 模块。
- 将保存逻辑从内存模拟升级为 localStorage。
- 增加动效模板导入 / 导出。
- 增加参数校验和异常提示。

第二阶段建议：

- 支持项目动效库持久化。
- 支持收藏和最近使用。
- 支持更完善的动效组合冲突检测。
- 支持 HTML + CSS 示例导出。
- 支持 JSON 配置重新导入。

第三阶段建议：

- 支持可视化 keyframes 编辑。
- 支持轻量时间轴。
- 支持公司动效库。
- 支持团队共享和版本管理。
- 支持 React / Vue 组件导出。
